# 1. メッセージング全般

## 1-1. 反応モード

| シーン | Botの動き |
|---|---|
| 1対1トーク（DM） | すべてのメッセージに反応 |
| グループ：通常メッセージ | サイレント（ログ保存のみ） |
| グループ：@Botメンション | メンション付きコマンドとして処理 |
| グループ：テキストプレフィックス（後述） | メンションなしでも反応 |

## 1-2. メンション代替プレフィックス（PC版LINE対応）

PC版LINEで `@` 候補にBotが出てこない問題を回避するための、テキストベースのプレフィックスです。
**文頭に**書けばメンション扱いになります。

| 入力例 | 動作 |
|---|---|
| `@WBG 5月施工で未完了ある？` | 進捗質問（半角@） |
| `＠WBG 田中さんに連絡しといて` | タスク登録（全角＠もOK） |
| `/WBG 未完了全部教えて` | 進捗質問 |
| `/wbg 仮タスク` | 仮タスク一覧（小文字でも可） |
| `秘書：館林店の進捗どう？` | 進捗質問（半/全角コロン可） |

通常会話（プレフィックスなし）は今まで通り無反応なので、グループの静けさは保たれます。

# 2. タスク管理

## 2-1. タスクの自動抽出・登録

LINEのメッセージから**担当者・内容・期日**をGeminiで抽出し、スプレッドシートに記録します。

**例：**
```
濱田さん 雨晴れクリニックの見積もりを4月25日までにお願いします
```
→ タスク管理シートに自動登録（担当：濱田 / 内容：見積もり提出 / 期日：4/25 / ステータス：confirmed）

## 2-2. プロジェクト保存先の選択（誤名寄せ防止）

メッセージから案件を**AIで自動確定する**のではなく、**候補を提示してユーザーに選ばせます**。
類似名のプロジェクト（例：「WOODBASE LP制作」「WOODBASE LINE秘書Bot」「WOODBASE 保守対応」）が誤って同一案件にまとめられるのを防ぎます。

### 動作フロー

1. グループが**特定案件に紐付け済み** → 従来通り即登録（明示的紐付けを尊重）
2. グループ**未紐付け** → LINEで **候補（最大4件）＋「🆕 新規プロジェクト作成」** を提示
3. 既存プロジェクトが**0件** → 新規プロジェクト名入力プロンプトへ直行

### 新規プロジェクト作成フロー

1. 候補UIで「🆕 新規プロジェクト作成」をタップ
2. Botが「保存先のプロジェクト名を次のメッセージでご入力ください（10分以内、「キャンセル」で中止）」と返信
3. ユーザーがプロジェクト名を送信
4. プロジェクト管理シートに新規追加 → 仮タスクを紐付けて保存 → LINEに登録結果を返信

## 2-3. 緊急度の自動判定 🚨

「至急」「急ぎ」「今日中」「最短」「GW前」などのキーワードを含むメッセージは自動で**緊急フラグ**が付きます。

- 確認UI：📋 → 🚨 表示
- 担当者通知：「🚨【至急】あなたへのタスク」
- スプレッドシート「緊急度」列に「高」と記録

**緊急キーワード：** 至急・急ぎ・急いで・ASAP・今すぐ・今日中・最短・緊急・GW前・GW中・明日まで・本日中

## 2-4. 誤検出フィルター

以下の定型メッセージはタスク登録対象から除外：

| パターン | 例 |
|---|---|
| 定型の承諾返事 | 了解しました・承知しました |
| お礼の言葉 | ありがとうございます |
| 挨拶 | お疲れ様でした・おはようございます |
| 確認報告 | 確認しました・見ました |

## 2-5. 重複タスク検出

同じ「グループ × 担当者 × 内容」のタスクが**48時間以内に既登録**なら、二重登録を自動スキップします。

## 2-6. タスク完了処理

DM または @Botメンションで「完了しました」「終わりました」等を送ると、AIが対象タスクを推定して完了扱いに更新します。

## 2-7. タスク照会（メンションコマンド）

| コマンド例 | 動作 |
|---|---|
| 残タスクは？ | 未完了タスクの一覧を表示 |
| 今週の予定は？ | スケジュール一覧を表示 |
| ○○案件の残タスクは？ | 案件別に絞り込み |

# 6. 進捗管理表 自然文クエリ

**別スプレッドシートの進捗管理表**を、LINEから自然文で質問できます。

## 6-1. シート連携設定

スクリプトプロパティで指定：
- `PROGRESS_SPREADSHEET_ID`：進捗管理スプシのID
- `PROGRESS_SHEET_NAME_OVERRIDE`：シート名（既定 `進捗管理表`）

設定ヘルパー：`setupProgressSpreadsheet(id, sheetName)`（GASエディタから1回実行）

## 6-2. 自動ヘッダー判定

ヘッダー行は1行目固定ではなく、`店舗名`/`図面チェック`/`施工月`/`依頼日` を含む行を自動検出します（タイトル行が上にあっても可）。

列名のファジーマッチで以下の言い回しに対応：

| ユーザーの言葉 | マッチする列 |
|---|---|
| 図面 / 図面確認 / 図面チェック | 図面チェック |
| 制作一覧 / 一覧入力 | 制作一覧入力 |
| 安陳 / 安陳入力 | 安陳入力 |
| 安陳見積 / 安陳見積もり | 安陳見積もり |
| スクショ / スクショ見積 | スクショ見積 |
| 金額 / 金額調整 / 金額確認 | 金額調整確認 |
| パイオニア / ﾊﾟｲｵﾆｱ（半/全角どちらも） | ﾊﾟｲｵﾆｱ見積 |
| 発注 | 制作発注 ＋ 発注書（両方） |
| 制作発注 | 制作発注 |
| 発注書 | 発注書 |
| 納品 | 納品日時 ＋ 納品場所（両方） |
| 納品日 / 納品日時 | 納品日時 |
| 完工 / 完工日 | 完工日 |
| 区分 / 美容 / 理容 | 理/美 |

## 6-3. 判定ルール

**完了とみなす値：**
- 各種 ◯/○/〇/◎/⚪/●/⬤/⭕/Ⓞ などの**丸記号**（Unicode 異体字を網羅）
- `丸` `まる` `マル` `済` `済み` `完了` `完` `ok` `OK` `done` `✓` `✔` `✅` `対応済`
- 日付列（依頼日・納品日時・完工日）：**値が入っていれば完了**

**未完了とみなす値（通知対象）：**
- 空欄（→「未入力」と表示）
- `△` `確認中` `×` `未` `保留` `対応中` `差戻し` `要確認` その他完了以外の文字列

## 6-4. 質問例

| LINEメッセージ | 動作 |
|---|---|
| 図面チェックできてない店舗ある？ | 図面チェック未完了の店舗一覧 |
| 依頼日入ってない店舗ある？ | 依頼日空欄の店舗一覧 |
| 5月の図面チェックまだのところ教えて | 5月施工×図面チェック未完了 |
| 確認中の店舗だけ教えて | 状態「確認中」のセルを抽出 |
| 6月施工で未完了ある？ | 6月施工分のすべての未完了 |
| 田中さん担当で未完了ある？ | 担当者「田中」で絞り込み |
| 〇〇店の進捗どう？ | 店舗名で絞り込み |
| 未完了全部教えて | 全項目の未完了一覧 |

## 6-5. 発火条件（誤検出回避）

- **タスク登録動詞**（お願い／しといて／送って／連絡しと／対応して 等）を含む文 → 進捗質問にしない（タスク登録に流す）
- 明示キーワード（未完了／まとめて／全部／一覧／進捗管理）→ 即発火
- フィルタ条件（列名／月／店舗／担当者／状態） ＋ クエリ動詞（ある？／教えて／まだ／どこ／?） ＋ 進捗シグナル → 発火

## 6-6. 表示仕様

- 表示上限 30件（超過分は店舗名のみリスト表示）
- 依頼日は結果に表示しない
- 件数が多い場合は月で分けて表示

# 7. 見積書 自動作成機能

工程管理表から、対象月の見積書シート（例：`26-5`）を自動で作成します。
**工程管理表と見積書のペアを「見積連携設定」シートに登録するだけで、複数案件に対応**できます（コード変更不要）。

## 7-1. LINEからの使い方

| 入力例 | 動作 |
|---|---|
| `東邦家具 5月分の見積書作って` | 表示名で指定 |
| `toho 5月分 見積作成` | 設定IDで指定 |
| `〇〇家具 2026年6月の見積書作成` | 年も明示可 |
| `見積設定一覧` | 登録済み設定の一覧を返信 |

該当なし：登録済み一覧と例文を返信。
複数該当：候補リストを返信して再送を促す。

## 7-2. 新しい案件を追加する手順

**管理用スプレッドシートの「見積連携設定」シートに1行追加するだけ**。コード変更は不要です。

### ① 「見積連携設定」シートに1行追加

| 列 | 入力例 |
|---|---|
| A: 設定ID | `client_b`（半角英数の案件キー） |
| B: 表示名 | `〇〇家具` |
| C: 工程管理表ID | `1AbCdEf...`（44文字） |
| D: 工程シート名 | `工程管理表`（タブ名） |
| E: 見積書ID | `1XyZ...`（44文字） |
| F: テンプレートシート名 | `原図`（テンプレタブ名） |
| G: 担当者表示 | `担当者：〇〇様` |
| H: 単価 | `5000`（製図時間あたり／空欄なら5000円） |
| I: 有効 | `TRUE` |
| J: 備考 | 任意メモ |

### ② スプシIDの取り出し方

スプシURLの `/d/` と `/edit` の間の **44文字** がID：
```
https://docs.google.com/spreadsheets/d/【ここがID】/edit#gid=0
```

### ③ 共有設定

工程管理表と見積書の両方を **`woodbasegroup@gmail.com`** に**編集者**として共有してください。

## 7-3. 月別シート名

- `2026年5月` → `26-5`
- `2026年12月` → `26-12`
- 西暦下2桁 + ハイフン + 月（**ゼロ埋めなし**）
- 同名シートが既存なら**上書きせず処理中止**＋LINEで通知
- 新規シートは**テンプレシートの右隣**に配置

## 7-4. 見積書の記載ルール

### 1ページ目（物件集計）

| 表示 | 内容 |
|---|---|
| 名称 | 【物件番号】物件名（新規〇枚／修正〇時間） |
| 数量 | 1 |
| 単位 | 式 |
| 単価 | 空欄 |
| 金額 | その物件の合計（製図時間×単価） |
| 備考 | 1行目：担当者表示／2行目以降：同上 |

### 2ページ目以降（明細）

物件ごとに見出し行＋以下の明細：

| 分類 | 数量 | 単位 | 単価 | 金額 |
|---|---|---|---|---|
| 修正（時間=1H） | 1 | H | （空欄） | 単価 |
| 修正（時間>1H） | 製図時間 | H | 単価 | 製図時間×単価 |
| 新規 | 枚数 | 枚 | （空欄） | 製図時間×単価 |

※ 新規も金額は **製図時間×単価** で計算。表示数量は枚数。

### 合計欄

- 小計 ＝ 全物件の金額合計
- 消費税 ＝ 小計 × 10%
- 税込合計 ＝ 小計 ＋ 消費税

1ページ目の上部（税込合計・消費税・合計）と下部（小計）に反映。

## 7-5. 自動処理できない行 → 要確認シート

以下は「見積要確認」シートに記録され、見積書には金額0で記載されます：

- 製図時間が空欄
- 分類が「検討のみ」「WBF」など未対応
- 分類が空欄

| 列 | 内容 |
|---|---|
| 作成日時 / 設定ID / 対象シート名 / 物件 / 図面名 / 分類 / 理由 | 自動記録 |

# 9. 自動実行スケジュール（毎日/毎週トリガー）

| 時間 | 処理 |
|---|---|
| 毎朝9時 | 期日リマインド・連絡漏れ検出 |
| 毎朝7時 | 案件サマリー（Docs）自動更新 |

## 9-1. 期日リマインド

| タイミング | 通知内容 |
|---|---|
| 期日3日前 | 「〇〇さん、期日が3日後です」 |
| 期日当日 | 「〇〇さん、本日期日のタスクがあります」 |
| 期日超過 | 「〇〇さん、期日を過ぎたタスクがあります」 |

## 9-2. 連絡漏れ検出

タスク登録から **3日経過しても未完了**の場合、自動でリマインド：
- 担当者DM：「このタスクの状況はいかがですか？」
- グループ：「○○さんへ：対応状況を確認中です」

一度送ったタスクには再送しません（スパム防止）。

## 9-3. 週次レポート

その週の **完了タスク数 / 未完了数 / 期日超過数** を社内グループに通知。

## 9-4. アーカイブ

完了タスクを「完了タスク」シートに移動して、タスク管理シートを軽く保ちます。
